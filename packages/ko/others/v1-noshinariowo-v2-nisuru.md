# v1의 시나리오를 v2로 이전하기

우즈 스튜디오에는 주황색을 기본 테마로 한 v1(구 버전 에디터)과 파란색을 기본 테마로 한 v2가 있습니다。

v1에서 만든 시나리오는 이 페이지에서 설명하는 절차를 따르면 v2 버전으로 공개할 수 있습니다。v1에서 할 수 없었던 것들이 v2에서는 많이 구현 가능해졌으므로 리메이크나 업데이트도 환영합니다。

## v2 이전 절차

### ① v1에서의 작업

※2024년 8월 6일 기준으로 운영 측에서 모든 시나리오에 대해 ①의 처리를 완료했습니다。②부터 실행해 주세요。

* 우즈 스튜디오 v1에서 이전하고 싶은 시나리오를 엽니다。
* 왼쪽 탭의 「작품을 제출」을 클릭합니다。
* 체크박스에 체크를 하고 「앱에 반영」을 누릅니다。

<figure><img src="../.gitbook/assets/image (143).png" alt=""><figcaption></figcaption></figure>

### ② v2에서의 작업

* 우즈 스튜디오 v2에 로그인합니다。
  * 우즈 앱에서 사용하고 있는 로그인 방식(전화번호·이메일 주소 등)으로 로그인합니다。
  * 우즈 스튜디오 v1에 등록되어 있는 이메일 주소로는 로그인되지 않으니 주의해 주세요。
* 우즈 스튜디오 v2의 시나리오 목록 화면에서, 방금 v1에서 앱에 반영을 누른 시나리오를 엽니다。
* 화면 상단의 탭들 중에서 「릴리스 관리」를 클릭합니다。
* 버전란의 「・・・」를 클릭하고, 「이 버전을 복원」을 클릭합니다。
* 성공하면 데이터 이전이 완료되므로, 오른쪽 상단의 「시나리오를 편집」이라는 버튼을 누릅니다。데이터가 정상적으로 표시되면 완료입니다。\
  ※ 시나리오가 찾을 수 없다고 표시되는 경우에는 [여기](https://docs.studio.uzu-app.com/qanda/qanda#v2shinariono)를 시도해 보세요。

<figure><img src="../.gitbook/assets/image (144).png" alt=""><figcaption></figcaption></figure>

### ③ 이전 신청 폼을 제출하기까지의 흐름

1. 위 설명을 참고하여 먼저 **v2 에디터에서 시나리오 복원**을 진행합니다。간단히 말하면 「v1 측에서 앱에 반영을 누른다」→「v2 측에서 복원한다」는 절차입니다。
2. 줄바꿈 등의 레이아웃이 일부 흐트러질 수 있으므로 **v2 에디터에서 조정**해 주세요。v2화에 따라 리딩 단계의 개선이나 여러 번 투표 개선 등을 진행하는 경우 해당 부분도 수정해 주세요。
3. 조정·수정이 끝나면, v2 에디터 오른쪽 상단의 버튼에서 **동작 확인**을 해 주세요。
4. 동작 확인에 문제가 없어 보이면, v2 에디터 오른쪽 상단의 버튼에서 「**최신 버전 생성**」을 눌러 주세요。
5. 생성에 성공하면 「릴리스 관리 화면」으로 자동으로 이동합니다。
6. 그곳에 있는 「v2 이전 신청」 버튼에서 신청을 진행해 주세요。

{% hint style="info" %}
그 후, 운영 측에서 시나리오를 확인하고, v2 버전을 일반 사용자에게도 공개할 수 있도록 처리합니다。공개를 알리는 공식 SNS 게시물은 기본적으로 20:00에 이루어집니다。
{% endhint %}

이전 신청 폼을 제출하고, 운영의 승인과 앱 상의 일반 사용자 공개가 완료되면 v2 이전은 완료됩니다。

### v2 버전을 테스트 플레이하고 싶은 경우

버전을 생성(앱에 반영)한 후, 작가 계정으로 우즈 앱 내의 시나리오 페이지를 보러 가면 「**최신 버전으로 이벤트 생성**」이라는 버튼이 나타납니다。그곳에서 이벤트를 생성하면, v2 버전의 테스트 플레이를 진행할 수 있습니다。

{% hint style="info" %}
참고로, 이 버튼은 작가 계정으로 로그인한 경우에만 보입니다。일반 사용자에게는 v1 버전을 공개한 채, 비공개로 v2 버전의 테스트 플레이를 할 수 있는 상태입니다。
{% endhint %}

## v1과 v2에서 다른 부분

* v2에서는 버튼 하나로 굵은 글자나 기울임(이탤릭), 제목 글자나 색상 글자로 바꿀 수 있게 되어 있습니다。

## v2로 이행함으로써 개선할 수 있는 것

### ◆ 대본 맞춰 읽기 페이즈

* v2에서는、**대본 맞춰 읽기 페이즈를 어디든지 몇 번이든 삽입할 수** 있게 되었습니다。
* v1에서 2회 이상의 대본 맞춰 읽기를 둘 때、캐릭터명「대사」처럼 텍스트로 대체하던 부분을 크게 개선할 수 있습니다。

* v1 시나리오에서 빈번했던「투표 전에 투표 대상을 하나로 좁히기 위한 긴 설명」을 둘 필요가 없어져、보다 자연스러운 형태로 투표할 수 있습니다。

* v2에서는、**시나리오 도중에도 조건에 맞춘 분기가 가능**하게 되었습니다。
* v1에서 자주 하던「줄바꿈으로 내용이 안 보이도록 만든 탭을 여러 개 배포하고 해당하는 것만 보게 하는」같은 상태를 크게 개선할 수 있습니다。

### ◆ 조사・단서

* v2에서는、**투표로 조사할 장소를 선택→해당하는 단서를 배포**하는 **조사**를 구현할 수 있게 되었습니다。
* 단서는「**전체 공개**」도「**양도**」도「**누군가에게만 공개**」도 할 수 있습니다。
* v1에서 자주 하던「링크를 탭→이미지 표시」로、플레이어의 양심에 맡긴 유사 조사를 크게 개선할 수 있습니다。

### ◆ 기타

* **캐릭터 텍스트를「어느 페이즈에 있을 때만 표시」할 수 있는** 덕분에、재독 금지 같은 기믹을 보다 자연스럽게 구현할 수 있습니다。
* 대본 맞춰 읽기 중의 특정 지문이나 대사를 계기로、**페이즈 도중에 BGM을 바꿀 수** 있게 되었습니다。

<figure><img src="../.gitbook/assets/image (7) (1).png" alt="" width="563"><figcaption></figcaption></figure>

* **이후의 페이즈를 숨길지 여부를 선택할 수** 있게 되었습니다。
  * 추가 토론이 있다는 것을 처음에는 보여주고 싶지 않은 경우 등에 유효합니다。
* **웹상에서의 동작 확인에서 BGM과 SE도 재생되게** 되었습니다。
* v2 시나리오에서는、**관전**이 가능합니다。
  * 작자가 관전자로 들어갈 수 있어、테스트 플레이가 더 쉬워졌습니다。
  * ※v1 그대로의 시나리오는 관전할 수 없습니다。* v1에서는 감상전 화면에서 모든 엔딩이 강제로 공개되고 있었지만, v2에서는**배포할 엔딩과 배포하지 않을 엔딩을 작가 측에서 제어할 수** 있게 되었습니다。
* 감상전 화면에서**진상 텍스트를 여러 탭으로 나눌** 수 있게 되었습니다。
  * 「진상 해설은 여기」「홍보는 여기」와 같은 사용이 가능해졌습니다。
* 「**시나리오 경향**」「**주의사항**」을 설정할 수 있게 되었습니다。
  * 「줄거리」나 「작가 코멘트」에 억지로 넣고 있던 주의사항이나 문장을 내용별로 제대로 분류하여 표시할 수 있게 되어 있습니다。

<figure><img src="../.gitbook/assets/image (8) (1).png" alt="" width="375"><figcaption></figcaption></figure>

우즈스튜디오 v2에는, 본 페이지에서 다 소개하지 못할 만큼 많은 기능이 있습니다。v1에서 다소 부자연스러운 형태로 구현되어 있던 것도, v2에서는 보다 자연스러운 상태로 개선할 수 있으니, 부디 당신의 시나리오도 v2판으로 이전해 보세요。

## 자주 묻는 질문

**Q. v2 이전을 위한 처리를 하면, 바로 v2판이 앱에 반영되어 버리나요？**

A. 그렇지 않습니다。이전과 조정이 끝나면 신청 폼에 응답해 주시면 됩니다。신청 폼의 응답을 운영이 확인하고 처리를 수행할 때까지, v2판이 앱에서 플레이 가능한 상태가 되지 않으니 안심하세요。

**Q. v2판의 테스트 플레이는 어떻게 하면 되나요？**

A. v1판을 앱에서 공개한 상태로, 작가만이 생성할 수 있는 이벤트로 비공개로 v2판의 테스트 플레이를 진행할 수 있습니다。작가는 관전자로서 참여하는 형태가 됩니다。
