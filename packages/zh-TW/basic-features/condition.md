# 全體共通：條件設定

角色的任務、階段、文本、線索、房間、遊戲流程等，幾乎所有的編輯畫面中都會出現「條件設定」。記住條件設定的種類和意義，設定適當的條件吧。

#### 條件群組

在部分條件設定中，也可以使用條件群組。使用條件群組可以設定**「且」與「或」的複合條件**。

對條件群組的更改是在正常添加條件後，從條件右上方顯示的「・・・」中進行的。在「・・・」中無法顯示更改建議的地方無法使用條件群組。

目前，條件群組最多可以設定到三層。

### 舊設計

※可設定的內容大致相同，但舊設計和新設計混合存在。

可設定的條件有以下8種類。若是「～時」，則可以使用「～不時」的否定版本條件。

| 項目                     | 基本內容                                                                                      | 其他                                               |
| ---------------------- | ----------------------------------------------------------------------------------------- | ------------------------------------------------- |
| 僅在特定階段中時          | 僅在該階段中顯示文本或房間，指定階段結束後消失。                                                   | 線索僅在該階段中可獲取，獲取後的線索會留在手中。 |
| 到達特定階段時       | <p>分發或顯示從該階段開始瞬間適用。<br>在任務達成判斷或分支判斷的語境中，表示是否通過該階段的意義。</p> | 分發或顯示會在該階段之後繼續。                            |
| 特定階段結束時          | 該階段結束後分發和顯示。                                                                    | 適合於閱讀的分發。                                   |
| 某選項最多得票時           | 可以選擇單獨最多和同票最多。主要用於分支處理。                                                             | 同票最多意味著「即使同票也視為該選項最多」。                     |
| 某角色投票給某選項時 | 字面意義。用於分支處理或分發信息的分配等。                                                                | 在推理類任務的達成判斷中也很活躍。                              |
| 某角色持有某線索時 | 字面意義。用於附加信息的分發判斷等。                                                                   | 在收集類任務的達成判斷中也很活躍。                              |
| 某角色在某房間時      | 可以將是否在特定通話房間中作為條件。                                                                   | －                                                 |
| 角色條件               | 用複選框來應用條件的角色
ターを選択できる。                                                             | －                                                 |

【and（かつ）】\
請以數學的意義來理解為「同時發生」或「兩者皆滿足」。這並不是像A＆B那樣單純的「和」。

【or（または）】\
指的是「只要其中一個發生即可」。
